# CONTRIBUTING.md

Thank you for contributing to Indobase! We're an open source Postgres development platform and we'd love your help. Here's how to get started:

[1. Getting Started](#getting-started)
[2. Issues](#issues)
[3. Pull Requests](#pull-requests)

## Getting Started

To ensure a positive and inclusive environment, please read our [code of conduct](CODE_OF_CONDUCT.md) before contributing. For help setting up the code in this repo, please follow [DEVELOPERS.md](DEVELOPERS.md). For the docs site, see [apps/docs/CONTRIBUTING.md](apps/docs/CONTRIBUTING.md) if you're changing documentation.

## Issues

If you find a bug, please open an issue so we can triage it.

- Please search [existing Issues](https://github.com/indobase/indobase/issues) before creating a new one.
- Include a clear description and steps to reproduce. Screenshots and URLs help.

## Pull Requests

We welcome pull requests. Before submitting:

- If you're fixing an issue, link your PR to it and check that no one else has already opened a PR for the same fix.
- For new features, consider opening a [Discussion](https://github.com/indobase/indobase/discussions) first so we can align on design.
- Use the PR template and provide clear context so we can review quickly.

**Pre-flight checks:**

- Run `pnpm build` (or `npm run build`) locally so the project builds.
- Ensure Prettier/lint checks pass.

Thank you for helping make Indobase better.
